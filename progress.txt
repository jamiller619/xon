## 2026-03-27 - US-023
- Implemented DRM detection in `drm.ts` for FairPlay (M4V/M4A), Widevine (MP4/M4V), and Adobe ADEPT (EPUB)
- Files changed: packages/server-core/src/drm.ts (new), src/drm.test.ts (new), src/orchestrator.ts (calls detectDrm for all new/changed files), src/index.ts (exports detectDrm)
- `detectDrm(filePath)` calls FairPlay, Widevine, and Adobe detectors in parallel via `Promise.all`; returns false on any error (non-blocking)
- FairPlay: opens M4V/M4A, walks ISO BMFF boxes at top level, returns true if `drms` box found
- Widevine: opens MP4/M4V/M4A/MOV, linearly scans first 64KB for `pssh` box, checks 16-byte System ID against Widevine UUID
- Adobe DRM: opens EPUB (ZIP), finds EOCD, iterates central directory for `META-INF/encryption.xml`, checks content for Adobe ADEPT namespace
- Orchestrator: `detectDrm` called for every new file before insert and every changed file before update; result stored in `drmProtected` column
- `drmProtected` column was already present in schema from US-007 — no migration needed
- 14 new tests in drm.test.ts (including ZIP builder helper); total 218 tests in server-core
- **Learnings for future iterations:**
  - `node:fs/promises` `open()` returns a `FileHandle` — mock it by providing `{ read, stat, close }` methods; `read` signature is `(buf, offset, length, position) => { bytesRead }`
  - ISO BMFF box walking: each box = size(4BE) + type(4ASCII); size includes the 8-byte header; iterate at `offset += size`
  - Widevine PSSH: linear scan is simpler than recursive box parsing — search for `pssh` bytes at positions [i+4..i+7], then read SystemID at [i+12..i+27]
  - ZIP central directory: EOCD at end of file gives `cdOffset` and `cdSize`; CD entries have 46-byte fixed header + variable filename/extra/comment; local file offset at CD entry byte 42
  - `EPUB` Adobe DRM: only need to check `encryption.xml` string content for the Adobe ADEPT namespace — no XML parsing required

## Codebase Patterns
- Drizzle migrations: `drizzle-orm/libsql/migrator` `migrate(db, { migrationsFolder })` applies SQL files; path uses `dirname(fileURLToPath(import.meta.url))` to work in both src/ and dist/
- Drizzle schema for SQLite: use `drizzle-orm/sqlite-core` (sqliteTable, text, integer); integer with `{ mode: "timestamp" }` for dates; integer with `{ mode: "boolean" }` for booleans; FK via `.references(() => table.col, { onDelete: "cascade" })`
- JSON columns in SQLite: store as text with `.default("[]")` and handle serialization in app code
- Monorepo with Yarn 1.22 workspaces: apps/*, packages/*, plugins/*
- drizzle-orm 0.40.x + @libsql/client 0.14.x: `drizzle(client)` from `drizzle-orm/libsql`; WAL mode via `client.execute("PRAGMA journal_mode=WAL")`
- drizzle.config.ts lives at package root (not src/); excluded from tsc since `include: ["src/**/*"]`; processed directly by drizzle-kit
- Biome import sorting: node: built-ins first, then third-party (alphabetical within groups)
- TypeScript with strict mode, NodeNext module resolution (ESM)
- Biome for linting/formatting (not ESLint/Prettier)
- Node version: v24.14.0 (despite PRD saying Node 22.x LTS, .nvmrc had v24.14.0)
- tsconfig.json uses composite: true for project references support
- Each new package needs: package.json (type:"module", exports with .js extensions), tsconfig.json (extends root), src/index.ts
- When adding a package, add its path to root tsconfig.json "references" array and run `yarn install`
- Use `yarn biome check <dir>` (not `yarn check`) for linting; `yarn check` validates workspace deps
- Node built-ins (e.g., node:path) require `@types/node` as a devDependency in each package that uses them
- Test framework: vitest (^2.1.0) — handles TypeScript/ESM natively without separate tsconfig; add per-package
- noUncheckedIndexedAccess: Record<string, T> indexing returns T | undefined — use `?? fallback` pattern
- Biome line width is 100; arrays/params exceeding this are auto-reformatted to multi-line on `biome check --write`
- Biome prefers `process.env.PORT` (literal key) over `process.env["PORT"]` (computed key) — use dot notation
- Hono: use `new Hono().basePath("/api/v1")` for prefixed routes; test with `app.request("/api/v1/...")` (no server needed)
- @hono/node-server: `serve({ fetch: app.fetch, port }, callback)` returns a Node.js HTTP server with `.close()`
- Packages with cross-package deps need `references` in their own tsconfig.json (not just root tsconfig.json)
- Hono sub-router params from parent routes (e.g., `:libraryId` in parent `/:libraryId/sources`) type as `string | undefined` — use `as string` cast; mount sub-routers with `router.route("/:parentParam/sub", makeSubRouter(db))`
- Drizzle boolean `{ mode: "boolean" }` returns JS `true`/`false`, not `1`/`0` — use `toBe(false)` in tests, not `toBe(0)`
- `tsc --build --noEmit` fails in TypeScript 5.9 with chained composite project references — use root `yarn typecheck` (`tsc --build`) for type checking
- Workspace packages already built (dist/ exists) don't need tsconfig.json references; adding them breaks `--noEmit` mode
- Biome `noImplicitAnyLet`: uninitialized `let` variables need explicit type annotations
- `.txt` is in the Documents media category — never a safe "non-media" test file; use `.log` or `.xyz`
- `vi.mock(module)` is hoisted; use `await import(module)` after mock to get typed mocked reference; flush microtasks with 2x `await Promise.resolve()`
- `exactOptionalPropertyTypes: true`: can't assign `T | undefined` to optional `T` property — use `if (typeof x === "string") obj.prop = x` pattern
- Biome `useLiteralKeys`: use dot notation (`obj.snake_case`) not bracket notation (`obj["snake_case"]`) for `Record<string, unknown>` access
- Mocking `node:child_process` spawn: create EventEmitter with `.stdout` EventEmitter, cast to `ReturnType<typeof spawn>`
- `music-metadata` `parseFile()` returns `{ common, format }` — `common.track`/`common.disk` always present with `.no: number | null`; `common.genre` is `string[]`; mock with `vi.mock('music-metadata', () => ({ parseFile: vi.fn() }))`, cast return as `never`
- Mock `node:fs/promises` with `vi.mock("node:fs/promises", () => ({ readFile: vi.fn() }))` alongside `node:child_process` mocks — both work independently per test file
- ZIP binary parsing: search for EOCD sig `0x06054b50` from end of file; CD entries at cdOffset each start with `0x02014b50` with uncompressed size at offset+24, filename length at offset+28 (46-byte fixed header + variable name/extra/comment)
- TAR parsing: 512-byte blocks; filename at 0-99, size in octal at 124-135, typeflag at 156 (0x30='0' or 0 = regular file); USTAR prefix at 345 when "ustar" magic at 257; `gunzipSync` from `node:zlib` to decompress .tgz before parsing
- `sharp` ESM default import; mock as `vi.mock("sharp", () => ({ default: vi.fn() }))`; use `.clone()` per size for batch-safe thumbnail generation; types bundled with sharp (no @types/sharp needed)
- Drizzle `ALTER TABLE ADD COLUMN` migrations generated cleanly via `yarn workspace @xon/server-core db:generate` after schema update
- When mocking sequential spawns (ffprobe then ffmpeg), use `mockImplementation` with `Promise.resolve().then(...)` for event emission — `mockReturnValueOnce` creates the proc before spawn is called, causing setTimeout to fire before listeners attach
- `Promise.resolve().then(...)` fires as a microtask after synchronous listener registration — ideal for mock EventEmitter event timing in async tests
- Hono `c.body()` accepts `Uint8Array<ArrayBuffer>` not `Buffer<ArrayBufferLike>` — use `new Uint8Array(buffer)` to convert Node.js Buffer for binary responses
- Biome `noNonNullAssertion`: avoid `arr[0]!` — use `const x = arr[0]; if (!x) return;` pattern instead
- `@hono/node-server` v1.x has no WebSocket support — use `ws` npm package with `noServer: true` + `server.on("upgrade", ...)` to handle WebSocket upgrades alongside Hono HTTP routes
- Module-level EventEmitter pattern (`eventBus`) decouples event producers from consumers; use `emitEvent(event: XonEvent)` wrapper for type safety
- `vi.useFakeTimers()` + `vi.advanceTimersByTimeAsync()` for testing setInterval scheduling without real time; remember `vi.useRealTimers()` in afterEach
- Mock `node:fs` with `vi.mock("node:fs", () => ({ watch: vi.fn() }))` to capture fs.watch callbacks; use `mockImplementation` to capture callback for debounce tests
- Biome `noNonNullAssertion` in test files: use `?.()` instead of `!()` for optional function calls (after `expect().toBeDefined()`)
- `fs.watch` (from `node:fs`, not `node:fs/promises`) for filesystem watching; supports `{ recursive: boolean }` options
- Frontend (apps/web): Vite 6 + React 19 + React Router 7 + Zustand 5; tsconfig uses `moduleResolution: "bundler"`, `lib: ["ES2022","DOM","DOM.Iterable"]`, `jsx: "react-jsx"`, `noEmit: true`; excluded from root tsconfig to avoid NodeNext/JSX conflicts
- Root tsconfig.json exclude must include `apps/web` to prevent root `tsc --build` from applying NodeNext module resolution to browser/JSX files
- CSS Modules with `noUncheckedIndexedAccess: true`: `styles.foo` returns `string | undefined`; use `styles.foo ?? ""` for direct className props or use template literals (always return `string`)
- Add `src/vite-env.d.ts` with `/// <reference types="vite/client" />` to enable CSS Module imports in TypeScript
- `hono/jwt` v4.12.x: `sign(payload, secret, "HS256")` and `verify(token, secret, "HS256")` — algorithm must be explicit or throws `JwtAlgorithmRequired`

- Auth test pattern: top-level `const AUTH = \`Bearer \${await signAccessToken(...)}\`` in ESM test files; add `Authorization: AUTH` to all test requests
- React Router v7 layout routes: `<Route element={<Layout />}>` wrapping child routes; `<Outlet />` renders child content inside the layout
